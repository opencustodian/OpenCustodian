# Security Policy

## Supported Versions

OpenCustodian currently maintains the latest schemas and documentation in the
`main` branch.

| Version | Supported |
| ------- | --------- |
| main    | ✅ |

## Reporting a Vulnerability

Please **do not open public GitHub issues** for security vulnerabilities.

Use one of the following private channels:

1. GitHub Security Advisories (preferred):
   - Open a private vulnerability report in this repository via the
     "Security" tab.
2. If advisory reporting is unavailable, contact maintainers privately through
   repository owner contact channels.

When reporting, include:

- A clear description of the issue
- Steps to reproduce
- Affected schema/file(s)
- Potential impact
- Suggested remediation (if known)

## Response Process

The maintainers aim to:

- Acknowledge receipt within 3 business days
- Provide an initial triage update within 7 business days
- Coordinate a fix and disclosure timeline based on severity

## Disclosure Policy

- We follow responsible disclosure.
- Reporters are asked to keep details private until a fix is available.
- Once resolved, maintainers will document impact and remediation in release
  notes or changelog entries.

## Scope

This policy applies to:

- JSON schemas in this repository
- Validation and generation scripts in `tests/scripts/`
- Associated documentation impacting secure usage
